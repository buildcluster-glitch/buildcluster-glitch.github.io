🚀1. 初めての方へ

(株)ビルドクラスタの業務アプリは スマホとPCどちらからでも 使えます。専用アプリのインストールは不要、ブラウザだけでOK。

① 入口URLをブックマーク

https://buildcluster-glitch.github.io/

このURLが全アプリへの入口です。タップすると工事タスクマネージャー / 在庫管理 などが選べます。

② スマホのホーム画面に追加（推奨）

i
iPhone (Safari): 共有ボタン ↑ → 「ホーム画面に追加」
A
Android (Chrome): 右上 ⋮ → 「ホーム画面に追加」
💡
ホーム画面から起動するとほぼアプリ感覚。タブ閉じても状態が保持されます。

③ 初回はインターネット必須

サーバーから初期データを取得する必要があるので、初回起動時だけWiFi or 電波の良い場所で。2回目以降はオフラインでも記録はできます（次回オンライン時に自動送信）。

⚙️2. 在庫管理アプリの初期設定

各端末で 最初に1回だけ やる作業です。設定が終わると、6人の入力データが共有スプレッドシートと自動連携します。

STEP 1. アプリを開く

入口ページ → 📦 在庫管理 をタップ

STEP 2. ⚙ 設定を開く

画面右上の ⚙ アイコンをタップ。

STEP 3. このデバイスを使う人を選ぶ

一番上の「このデバイスを使う人」セクションで 自分の名前 をタップ。 これで以降の記録に自動で名前が入ります。

ℹ️
他の人の端末で操作する時は、設定で名前を切り替えてください。共用PCの場合は記録時に都度選択も可。

STEP 4. GAS Web App URL を貼り付け

「クラウド同期」セクションの URL 入力欄に、以下を貼り付け → 保存 → 接続テスト:

https://script.google.com/macros/s/AKfycbwMmI7cBmTzbL7C2sjZDX_3kWc4G-fQyLrzYJeD8Ya1Sgsv2HywhKzkd0RgZ7IPqk0UFg/exec

「接続テスト」を押して ✓接続OK items:38 / logs:N / orders:N のような表示が出れば成功。

⚠️
URL は前後にスペースや改行が入らないように。コピーしてそのまま貼り付けが確実です。

STEP 5. サーバーからデータ取込

「初回セットアップ（サーバーから取込）」ボタンを押す → 確認ダイアログで「OK」 → 取り込み完了。

一覧画面に37+件の品目が表示されればセットアップ完了！

🎉
これで準備完了。あとは普通に操作するだけで自動的に他の端末にも反映されます。

📦3. 在庫管理アプリの使い方

画面構成（4タブ）

タブ用途
一覧在庫数を見る・検索・カテゴリ&置き場フィルタ・在庫アラート確認
発注中注文済みで未納の品目を管理。入荷処理もここから
棚卸実数を入れて記録在庫と突き合わせ・差分の理由記録
ログ全操作の履歴。誰がいつ何をしたか追える

消費を記録する（一番よく使う）

1
右下 ＋ ボタンをタップ
2
− 消費 をタップ
3
カテゴリチップで絞り込み → 品目をタップ
4
数量 / 担当者（自動選択済み） / 使用現場 / 号室 / 備考 を入力
5
記録する ボタン → 在庫が減って自動同期

補充を記録する（届いた時）

消費と同じく FAB ＋ から ＋ 補充 を選ぶ。数量を入れて記録するだけ。

発注を記録する（注文した時）

1
FAB ＋ → 📦 発注を記録
2
品目選択 → 数量・注文先（自動入力）・予定納期（リードタイムから自動計算）
3
発注を登録 → 発注中タブに表示される
ℹ️
2〜3日で届く品なら発注登録はスキップしても可。届いた時に「補充」記録だけで完結。

入荷処理（届いた発注を在庫に反映）

1
発注中 タブを開く
2
対象行の 📥 入荷 ボタン
3
入荷数量（部分入荷もOK）・受領者・入荷日 を入力
4
入荷処理する → 在庫が増え、発注中から消える（全量入荷時）
💡
入荷時、注文日→入荷日の リードタイムを自動学習。次回発注時の予定納期が精度上がる。

週次棚卸

1
棚卸 タブを開く
2
置き場チップで 🏠倉庫 or 🚪外物置 を選ぶ（場所別に作業）
3
各品目の「実数」欄に数えた値を入力 → 差分が黄色で表示
4
差分ありの行に 理由 を選択（破損 / 使用記録漏れ / 紛失 など）
5
記録して在庫修正 → 在庫が実数に合わせて修正 + 監査ログ残る

品目の追加・編集・削除

  • 追加: 一覧タブの検索バー横「＋ 品目」ボタン
  • 編集: 各カードの右上「✎」アイコン or 詳細画面の「編集」ボタン
  • 削除: 編集モーダル内の赤い「この品目を削除」ボタン

月次レポート

ヘッダー右上の 📊 ボタンから。月セレクタで前後月に移動できる。

  • 消費記録数・総消費数・推計金額
  • 品目別TOP10（バー付き）
  • 担当者別 / カテゴリ別 / 物件別の集計
  • 発注件数・入荷件数・平均リードタイム

ログの便利機能

  • 品目名タップでその品目の詳細画面（履歴一覧）にジャンプ
  • ↩ 取消ボタンで消費/補充を取消（在庫も逆方向に戻る）
  • ✕ 削除ボタンでログだけ削除（在庫はそのまま）
  • 種別フィルタ・メンバーフィルタで絞り込み

☁️4. クラウド同期の仕組み

データの流れ

📱 スマホA 📱 スマホB 💻 PC │ │ │ │ 消費 -2 │ 補充 +1 │ 棚卸 │ adjustStock │ adjustStock │ setStock ↓ ↓ ↓ ┌──────────────────────────────────┐ │ ☁ GAS Web App (LockServiceで順次処理) │ │ ↓ │ │ 📊 Google スプレッドシート (在庫管理DB) │ │ ・items ・logs ・orders ・members │ └──────────────────────────────────┘ ↑ ↑ ↑ │ 30秒ごと自動pull │ │ ↓ ↓ ↓ 全端末で 常に最新の同じ数値が見える

同期のタイミング

タイミング動作
アプリ起動時サーバーから最新取得
タブから戻った時即座に最新取得
操作（消費・補充等）した時1.5秒後に送信 + 取得
30秒経過（画面表示中）自動で取得
☁ アイコンをタップ即座に送信 + 取得

☁ アイコンの状態

表示意味
☁ (白)正常・アイドル
☁ (回転)同期中
☁ (緑)直近の同期成功（2秒で消える）
☁ (赤)直近の同期失敗（次の自動同期 or タップで再試行）
☁ + 黄色バッジ「N」未送信の変更が N 件残ってる

同時操作への対応

6人が同じ時刻に同じ品目を操作しても データが消えない 仕組みになってます:

  • 差分送信: 「stock=3にしろ」ではなく「-2引け」と命令を送る
  • atomic処理: サーバー側でロックを取って順番に適用
  • 冪等性キー: ネットワーク不調で再送されても重複適用しない

オフラインでも記録できる

電波の悪い現場でも消費記録は普通にできます。ローカルに保留され、電波が戻ると 自動で送信。 ヘッダーの☁に黄色バッジで未送信件数が見えます。

🛠5. 困った時（FAQ）

Q1. ☁が赤くなってる

同期失敗のサイン。30秒以内に自動で再試行されるので待つだけでOK。急ぐ場合は ☁をタップ で即再試行。

それでも赤いままなら:

  • 電波・WiFi が通じているか確認
  • ⚙設定 → 「接続テスト」を実行 → エラーメッセージ確認
  • URLが正しく入っているか確認（前後の余白に注意）

Q2. 数が合わない

まず ☁ で同期 → それでもズレてたら 棚卸タブで実数を入力 → 理由「カウントミス」or「使用記録漏れ」を選ぶ。これで在庫が実数に合います。

Q3. 間違って消費・補充を記録した

ログタブでその記録を探す → 右の ↩ 取消 ボタン → 在庫も自動で戻る。

Q4. 古いデータが見える

Ctrl+F5（PC）or タブ閉じて開き直し（スマホ）で強制再読み込み。それでもダメなら ⚙設定 → 「初回セットアップ（サーバーから取込）」で上書き。

Q5. 別の人の端末で見ても自分の記録が見えない

自分の端末で ☁ をタップして送信完了を確認 → 相手の端末で ☁ をタップ or 30秒待つ。

Q6. 自動同期を止めたい

⚙設定 → 「自動同期」のチェックを外す。手動で ☁ タップ時のみ同期。

Q7. 担当者を間違えて記録した

現状ログの「担当者だけ」を後から変更する機能はないので、 ↩ 取消 → 正しい担当者に切り替えて再記録、が確実です。

Q8. 新メンバーが入った/辞めた

⚙設定 → メンバーセクションで追加/削除。全端末で同期されます。
